网络设备 Equipment:
    网卡(Interface): 
        运行位置: 物理层（电信号发送） + 数据链路层（链路建立和拆除，帧定界同步顺序差错控制等）
    
    中继器: 
        运行位置: 物理层
   

    集线器(HUB): 
        运行位置: 物理层
        n 个端口，任何端口接收数据，都广播到所有其他端口
        
    网桥(Bridge): 
        运行位置: 数据链路层
        一般只有两个端口，在两个端口间进行转发
        
        
        网桥是 硬件网络协议翻译器
        两个网络之间 协议不同，一个是 TCP/IP 一个是 令牌环，两个网络之间需要传输数据，就需要网桥进行翻译 转发。
        @attention 现在的操作系统之间为了互相交流，支持更多的协议，操作系统自己就可以是网桥，现在网桥这个概念已经淡出了
        @attention 网桥主要由软件实现，交换机主要由硬件实现!
    
    以太网(局域网)交换机(Switch): 
        运行位置: 数据链路层
        n 个端口，一个端口接收到数据，
        如果目标地址在 缓存中，则转发给 对应端口，否则 广播
        
        @attention 交换机使用 MAC 地址 和  ARP 协议
    
    路由器/三层交换机(Router)：
        运行位置: 网络层
        @attention 路由器使用 IP 地址 
     
    网关/四层交换机(Gateway)：
        运行位置: 传输层及以上
        可以将一个网络的 IP 数据包，发送给另一个网络的主机的设备，所以 网关 在各个网络中，都有一个属于该网络的 IP 地址
        @attention 即使
            比如有网络A和网络B，
            网络A的IP地址范围为“192.168.1.1~192. 168.1.254”，子网掩码为255.255.255.0；
            网络B的IP地址范围为“192.168.2.1~192.168.2.254”，子网掩码为255.255.255.0。
            在没有路由器的情况下，两个网络之间是不能进行TCP/IP通信的，
            即使是两个网络连接在同一台交换机(或集线器)上，TCP/IP协议也会根据子网掩码(255.255.255.0)判定两个网络中的主机处在不同的网络里。
            而要实现这两个网络之间的通信，则必须通过网关。
        

网卡：
    工作模式：
        广播模式（Broad Cast Model）：接收 IP packet 目标地址为 广播地址(0Xffffff) 的数据包
        多播模式（MultiCast Model）：接收 目标地址 为 多播地址 的数据包，无论是否在 多播组 内
        直接模式（Direct Model）：只接收目地址是自己 Mac 地址的帧
        混杂模式（Promiscuous Model）：网卡接收所有的流过网卡的帧，抓包程序就是在这种模式下运行的
